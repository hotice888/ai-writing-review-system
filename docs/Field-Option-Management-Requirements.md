# 字段选项管理模块需求设计文档

## 一、功能概述

字段选项管理模块用于管理系统中各种下拉框、级联选择框等的字段和选项配置，支持简单字段和多级级联字段（如省市区）的灵活配置。

## 二、数据模型设计

### 2.1 字段表（field_options）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | UUID | 主键 | PRIMARY KEY |
| field_name | VARCHAR(100) | 字段名称 | NOT NULL |
| field_code | VARCHAR(100) | 字段标识 | NOT NULL, UNIQUE |
| status | VARCHAR(20) | 状态 | NOT NULL, 默认'enabled'，值：enabled/disabled |
| parent_field_id | UUID | 上级字段ID | 外键关联field_options(id) |
| field_level | INT | 字段级别 | NOT NULL, 默认1 |
| description | TEXT | 描述 | |
| created_at | TIMESTAMP | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 更新时间 | DEFAULT CURRENT_TIMESTAMP |

### 2.2 选项表（field_option_items）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | UUID | 主键 | PRIMARY KEY |
| field_id | UUID | 字段ID | NOT NULL, 外键关联field_options(id) |
| option_text | VARCHAR(200) | 选项Text | NOT NULL |
| option_value | VARCHAR(200) | 选项Value | NOT NULL |
| status | VARCHAR(20) | 状态 | NOT NULL, 默认'enabled'，值：enabled/disabled/archived/deleted |
| display_order | INT | 显示序号 | NOT NULL, 默认0 |
| parent_option_id | UUID | 上级选项ID | 外键关联field_option_items(id) |
| created_at | TIMESTAMP | 创建时间 | DEFAULT CURRENT_TIMESTAMP |
| updated_at | TIMESTAMP | 更新时间 | DEFAULT CURRENT_TIMESTAMP |

## 三、功能详细设计

### 3.1 页面结构

#### 3.1.1 列表页
包含两个Tab页签：
- **字段列表Tab**：显示所有字段的基本信息
- **选项列表Tab**：显示各级字段的所有选项（仅查看和状态切换）

#### 3.1.2 详情页
用于添加和编辑字段及选项（无单独的选项详情页）

---

### 3.2 字段列表Tab页

#### 3.2.1 表格列
- 字段名称
- 字段标识
- 状态（启用/禁用）
- 字段级别
- 上级字段
- 描述
- 操作列

#### 3.2.2 操作列功能
- 状态切换按钮（一键切换启用/禁用）
- 编辑按钮
- 删除按钮

---

### 3.3 选项列表Tab页

#### 3.3.1 表格列
- 所属字段
- 选项Text
- 选项Value
- 状态（启用/禁用/封存/逻辑删除）
- 显示序号
- 上级选项
- 操作列

#### 3.3.2 操作列功能
- **仅提供状态切换按钮**（启用/禁用/封存切换）
- **不提供添加、编辑、删除按钮**

---

### 3.4 字段详情页（添加/编辑）

#### 3.4.1 基础信息区
- 字段名称（必填）
- 字段标识（必填，唯一）
- 状态（启用/禁用，默认启用）
- 上级字段（下拉选择，可选）
- 字段级别（数字输入，默认1级，自动根据上级字段计算）
- 描述（文本域，可选）

#### 3.4.2 选项设置区（核心功能）

##### Tab页签结构
- 第一个Tab：显示当前字段，Tab名称显示字段层级（如"一级"、"二级"等）
- 子字段Tab：如果有子字段，自动添加对应的Tab页签
- 添加子字段按钮：在Tab栏右侧提供"+"按钮，点击添加下级字段
- **无"编辑多项"按钮**

##### 每个Tab的内容
- 该字段的选项列表（表格形式）
- 添加选项按钮
- 删除选项按钮
- 刷新序号按钮
- 批量删除按钮
- **直接在表格中编辑**（双击或点击编辑图标进入编辑模式）
- **无批量启用/禁用/封存**，状态切换在每行单独操作
- 拖拽排序调整显示序号
- 批量导入选项

##### 选项列表表格列
- 复选框
- 序号
- 选项名称（可直接编辑）
- 选项Value（可直接编辑）
- 显示序号
- 状态（下拉选择：启用/禁用/封存，切换直接保存）
- 操作列（删除按钮）

#### 3.4.3 层级式维护（推荐方式）
- 当前字段是上级时，在其选项中选择某个选项
- 点击"+"添加子字段Tab
- 在子字段Tab中维护下级选项
- 下级选项自动关联到选中的上级选项
- 支持多级级联（省→市→县...）

## 四、API接口设计

### 4.1 字段相关接口

#### 4.1.1 获取字段列表
```
GET /api/admin/field-options
查询参数：
- page: 页码
- pageSize: 每页数量
- keyword: 搜索关键词（可选）
- status: 状态筛选（可选）

响应：
{
  code: 200,
  message: 'success',
  data: {
    list: [...],
    total: 0,
    page: 1,
    pageSize: 20
  }
}
```

#### 4.1.2 获取字段详情
```
GET /api/admin/field-options/:id

响应：
{
  code: 200,
  message: 'success',
  data: {...}
}
```

```
  data: {...}
}
```

## 六、使用场景举例

### 场景1：简单字段 - 业务类型
1. 创建字段：
   - 字段名称：业务类型
   - 字段标识：business_type
   - 字段级别：1
2. 添加选项：
   - A类型 / A标识
   - B类型 / B标识

### 场景2：级联字段 - 省市区
#### 方式一：分别创建字段
1. 创建"省"字段（1级）
   - 添加选项：广东省、湖南省...
2. 创建"市"字段（2级，上级=省）
   - 添加选项：广州市、深圳市...（关联到广东省）
3. 创建"县"字段（3级，上级=市）
   - 添加选项：天河区、南山区...（关联到广州市）

#### 方式二：层级式维护（推荐）
1. 创建"省"字段（1级）
   - 添加选项：广东省
2. 选中"广东省"，点击"+"添加"市"子字段
   - 在"市"Tab中添加：广州市、深圳市
3. 选中"广州市"，点击"+"添加"县"子字段
   - 在"县"Tab中添加：天河区、越秀区
