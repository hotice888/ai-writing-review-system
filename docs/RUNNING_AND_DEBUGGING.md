# 运行和调试指南

## 1. 运行服务

### 后端服务
```bash
# 进入后端目录
cd backend

# 安装依赖（首次运行时）
npm install

# 启动服务
npm start
```
后端服务默认运行在 http://localhost:3000

### 管理端前端
```bash
# 进入管理端前端目录
cd adminFrontend

# 安装依赖（首次运行时）
npm install

# 启动开发服务器
npm run dev
```
管理端前端默认运行在 http://localhost:5174

### 用户端前端
```bash
# 进入用户端前端目录
cd homeFrontend

# 安装依赖（首次运行时）
npm install

# 启动开发服务器
npm run dev
```
用户端前端默认运行在 http://localhost:5173

## 2. 在 VS Code 中调试

### 配置文件
已创建 `.vscode/launch.json` 文件，包含以下调试配置：

1. **Debug Backend** - 调试后端服务
2. **Debug Admin Frontend** - 调试管理端前端
3. **Debug Home Frontend** - 调试用户端前端

### 如何使用调试配置：

1. 点击 VS Code 左侧的 "运行和调试" 图标（或按 Ctrl+Shift+D）
2. 在顶部的下拉菜单中选择要使用的调试配置
3. 点击绿色的 "开始调试" 按钮（或按 F5）

### 调试后端服务：
- 选择 "Debug Backend" 配置
- 点击开始调试
- 后端服务会在调试模式下启动
- 你可以在代码中设置断点，查看变量值等

### 调试前端服务：
- 确保前端服务已经启动（使用 `npm run dev`）
- 选择对应的前端调试配置
- 点击开始调试
- 会打开一个新的 Chrome 窗口，连接到调试器
- 你可以在 VS Code 中设置断点，查看前端代码的执行情况

## 3. 查看日志

### 后端日志：
- 后端服务启动后，在终端中会显示详细的日志信息
- 包括数据库初始化、菜单创建、服务器启动等信息

### 前端日志：
- 前端服务启动后，在终端中会显示 Vite 的开发服务器日志
- 浏览器的开发者工具（按 F12）中的控制台也会显示前端运行时的日志和错误信息

## 4. 常见问题和解决方案

### 端口被占用：
- 运行 `node kill-ports.js` 可以终止占用 3000、5173、5174 端口的进程
- 或者手动修改 `.env` 文件中的端口配置

### 数据库连接失败：
- 确保 PostgreSQL 服务正在运行
- 检查 `backend/.env` 文件中的数据库连接配置是否正确
- 确保数据库已经创建（可以运行 `node setup-db.js` 初始化数据库）

### 前端无法连接到后端：
- 确保后端服务正在运行
- 检查前端 `.env` 文件中的 API 基础 URL 配置是否正确
- 检查浏览器的网络请求，查看是否有 CORS 错误

### 权限问题：
- 确保登录用户有相应的权限访问菜单和功能
- 可以使用超级管理员账号登录（默认账号：admin@example.com，密码：123456）

## 5. 开发流程

1. 启动后端服务
2. 启动前端服务
3. 在 VS Code 中进行开发和调试
4. 保存代码后，前端会自动热更新
5. 后端代码修改后需要重启服务才能生效

- 超级管理员：admin@example.com / 123456
- 普通用户：通过注册页面注册
